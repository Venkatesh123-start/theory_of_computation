# Contributing to Theory of Computation

Thanks for your interest in contributing! This project welcomes improvements, corrections, new examples, and exercises.

How to contribute

1. Fork the repository and create a feature branch from `main` using a descriptive name (e.g., `feature/dfa-examples`).
2. Make your changes in a clear, focused commit.
3. Submit a pull request describing the changes and linking any related issues.

Guidelines

- Keep changes small and focused.
- Include examples or tests when adding new content (worked examples, diagrams, or step-by-step solutions).
- Cite sources when adding material from external references (textbooks, papers, websites).
- Use clear, concise language and prefer examples that can be validated manually.

Code style and formatting

- Markdown files: use fenced code blocks for code or automaton descriptions.
- Filenames: use lowercase and underscores, e.g., `finite_automata/example_dfa.md`.

Reporting bugs and requesting features

- Open an issue with a descriptive title and steps to reproduce (for bugs) or a clear explanation of the requested feature.

License and copyright
